# 2026-04-28 — Plugin scaffolded from template

Bootstrapped `obsidian-book-exporter` from `obsidian-plugin-template`. Removed inherited template state (CHANGELOG, versions.json, manifest, package.json identity, `init` script and `TEMPLATE_USAGE.md`).

## Decisions

- Desktop-only plugin (`isDesktopOnly: true`) — needs `pandoc` and `ebook-convert` over `child_process`.
- TOC parsing uses level-2 headings (`Front Matter`, `Chapters`, `Back Matter`) + bulleted wikilinks. Configurable.
- Compiler strips frontmatter, drops the first H1 of each note, demotes remaining headings, rewrites Obsidian-only syntax (callouts, embeds, comments).
- MOBI is produced by converting the EPUB intermediate via Calibre.
- Pandoc is fed metadata via a generated YAML file rather than CLI flags to avoid escaping issues.
- Settings stay immutable — Immer `updateSettings(draft => ...)`.

## Open questions

- Pandoc 3.x default `--top-level-division` behavior with `--toc-depth=2` — verify chapter-vs-section heading mapping inside an EPUB once pandoc runs against a real book.
- For PDF cover: pandoc supports `cover-image` via metadata but rendering depends on the engine (xelatex needs a custom template). Out of MVP — revisit.
- Cycle detection in note embeds (`![[Note]]`) is currently absent — the manuscript compiler does not recurse into embeds; it only resolves images. Decision: leave note embeds as inline text expansion for a follow-up.
