# 2026-07-17 — Two-phase release workflow (attestation binds to tag commit)

Ported the fixed release workflow from obsidian-remarkable-sync. The old workflow built and attested artifacts at the triggering commit, then created the version-bump commit and tag — so the build-provenance attestation was bound to a different commit than the release tag, and the community-catalog review rejected releases ("attestation failed cryptographic verification"). New design: a `prepare` job (dispatch on a branch) bumps/commits/tags (lightweight tag) with a build+test gate, then re-dispatches the workflow at the tag ref with `publish=true`; a `publish` job (dispatch-at-tag or bare tag push) builds, attests (`continue-on-error` removed), and creates the release with `github.sha` equal to the tag commit. Design proven live on remarkable-sync 1.5.0 (cert sha == tag commit).
