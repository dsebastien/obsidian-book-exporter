# 2026-06-24 — Changelog: stop mangling `@`-prefixed tokens into bogus links (issue #45)

Fix for [issue #45](https://github.com/dsebastien/obsidian-book-exporter/issues/45): the changelog/release-notes generator turned code-ish `@token`s in commit subjects into self-referential GitHub profile links. The 0.0.8 notes rendered `[@citations](https://github.com/citations)` from the commit subject `fix(typst): neutralise stray @citations …`. `conventional-changelog` treats a leading `@word` as a GitHub @mention and links it to `https://github.com/<word>`, which is wrong for non-user tokens (Typst `@citations`, a `@set` directive, etc.).

## Changes

- `scripts/generate-changelog.ts`
    - New exported helper `neutraliseMentionLinks(text)`: rewrites auto-generated mention links where the displayed handle equals the URL path (`[@token](https://github.com/token)`) into inline code (`` `@token` ``). GitHub will not re-autolink text inside backticks, so the token renders as plain text. Real `issues/`, `commit/`, and `compare/` links are untouched because their link text never equals the bare URL path.
    - `generateChangelog()` runs new entries through the helper before writing `CHANGELOG.md`.
    - `syncToDocsReleaseNotes()` runs the helper too, so previously generated entries get cleaned on the next docs sync.
- `scripts/generate-changelog.spec.ts` — coverage for the helper: rewrites a bogus self-mention, handles `http`/multiple mentions per line, leaves real issue/commit links untouched, and leaves mention links whose text differs from the URL path untouched.
- `CHANGELOG.md` + `docs/release-notes.md` — fixed the already-mangled 0.0.8 `@citations` line in place (minimal one-line edit each; no full regeneration to avoid unrelated bullet churn).

## Design notes

- Equality between link text and URL path is the signal that the link was machine-generated from a token rather than an intentional attribution link. This keeps genuine maintainer mentions (which point at a real profile that happens to differ, or are written deliberately) working.
- Chose backticks over bare text so the `@` reads as code (matching the project's convention of wrapping code-ish tokens) and is robustly protected from GitHub autolinking.
- Did not touch the separate `[#set](…/issues/set)` artifact visible in older entries; that is a different (issue-link) mangling and out of scope for this issue.

## Verification

- `bun run tsc` — clean.
- `bun test scripts/generate-changelog.spec.ts` — 7 pass / 0 fail.
- `bun run lint` — clean (`--max-warnings 0`).
- `bun run format` — no changes (sources already conform).
- End-to-end: ran `syncToDocsReleaseNotes()` against the real `CHANGELOG.md` and confirmed `[@citations](https://github.com/citations)` becomes `` `@citations` `` in `docs/release-notes.md`.

### [#35](https://github.com/dsebastien/obsidian-book-exporter/issues/35) — Error 43 classifier

### [#36](https://github.com/dsebastien/obsidian-book-exporter/issues/36) — HTML engines

Two steps. First **dropped** `weasyprint` + `wkhtmltopdf` from `PdfEngine` (they were offered but the matter transitions and cover were Typst/LaTeX-only, so output was silently wrong). Then **re-added weasyprint as a first-class engine** (wkhtmltopdf stays out):

- `manuscript-compiler.ts` — `BODY_MATTER_OPEN` also emits a `{=html}` `<div class="body-matter">` wrapper (closed by `BODY_MATTER_CLOSE` at document end); `buildHtmlCss` generates a stylesheet (front matter lower-roman via default `@page`, named `@page body` decimal with the reset on `@page :nth(1 of body)`, full-bleed `.book-cover`, page setup); `buildHtmlCover` emits the before-body cover fragment.
- `pandoc-runner.ts` — weasyprint gets `--css` and `--include-before-body`; `pushPageSetupArgs` skips it (its setup is in CSS).
- Re-added `weasyprint` to `PdfEngine`, the parser validator, and the settings dropdown.

### Preflight probe — every engine

### Documentation

- User guide (`docs/`): Troubleshooting (README), engine + matter-transition + page-setup coverage, the ribbon/menu entry points, error-hint debugging, the missing font/page-setup settings and `book_export` keys.

## Design notes

- **weasyprint page-counter reset.** weasyprint ignores `counter-reset: page` on document elements — it only honours a reset declared in an `@page` rule (`layout/page.py:_standardize_page_based_counters`). Named pages also don't propagate to sibling boxes, so the body content is wrapped in `.body-matter` and the reset is on `@page :nth(1 of body)`. Verified empirically (cover unnumbered → front matter i/ii → body reset to 1, A5).
- **wkhtmltopdf stays unsupported.** WebKit-based, abandoned, no usable `@page` counter support — it can't restart body-matter numbering, which is the whole point.
- **Preflight now probes LaTeX engines too.** The earlier rationale for skipping them (false negatives from a stripped `$PATH`) is moot now that the probe runs under `buildSpawnEnv`'s augmented PATH. `<bin> --version` checks binary existence; a missing TeX _package_ is a different failure class that surfaces at export time via the #35 classifier.
